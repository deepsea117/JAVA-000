写一段对于不同GC的总结：
串行GC：每次GC执行清除垃圾效果好，但每次发生时触发STW,已淘汰
并行GC：jdk6/7/8默认GC，清除垃圾效果好，每次执行时也触发STW，适用于高并发、高吞吐的应用场景
CMS GC: 执行频率高，清除垃圾效果一般，适用于低延迟应用场景
G1 GC：jdk9以后默认GC,集成并行GC和CMS GC的优点，适用于内存空间比较充裕的系统
